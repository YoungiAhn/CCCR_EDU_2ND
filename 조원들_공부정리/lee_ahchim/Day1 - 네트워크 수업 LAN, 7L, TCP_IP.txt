프로토콜 : 각 통신 계층에서 사용하는 통신 규칙/규약. L2 Eth, L3 IP, L4 TCP/UDP 등등.
OSI 모델 : 국제표준화기구 OSI에서 제공하는 네트워크 통신용 7계층 참조 모델. 네트워크 프로토콜 통신의 데이터 전송과 변환 처리 과정을 7개 계층으로 제시한 모델/아키텍처. 계층별로 독립성을 가지도록 잘 나누어놨기 때문에 장애 발생 시 대처나 처리에 용이하다는 장점이 있음.
TCP/IP 모델 : 패킷 통신 방식의 인터넷 프로토콜을 기반으로 만든 4계층 모델. 실제 인터넷 통신에서 사용하는 패킷에서 참조되는 계층만 모델에 포함되어 있으며, 7계층에 비해 계층을 4계층으로 단순화했기 때문에 보안이나 네트워크 분야에서는 꽤 실용적. 
캡슐화 / 역캡슐화 : 시스템에서 다른 시스템으로 데이터를 전송하기 위해 각 계층을 지나게 됨. 그 과정에서 해당 계층의 주소 혹은 에러 검출 등의 정보가 담겨있는 헤더를 각 계층에서 데이터 앞단에 붙여 데이터를 패키징하는 과정을 캡슐화 / 캡슐화된 데이터를 받고 나서 해당 헤더를 참조하여 (내 데이터/패킷이 아니면 같은 계층의 장비에 넘겨주거나 버리고), 내 정보가 맞다면 헤더를 떼고 다음 계층 장비로 넘겨주는 과정을 역캡슐화라고 함.
	(읽어보셈 - http://www.ktword.co.kr/abbr_view.php?m_temp1=379)
헤더 : 캡슐화 과정에서 자신의 계층 특성을 담은 제어정보(주소,에러제어)가 담겨 있는 데이터를 앞에 붙이는데 그것을 가리킴. 뒤에 붙이면 Tail 혹은 Trailer라고 부름, 헤더가 주요 사용처.


OSI 7계층 정리
L1 피지컬 : 실제 네트워크의 물리적 형태를 규정(여기서는 토폴로지라고 함)해서 데이터의 전기적 흐름에 따라(01011011...) 전송하는 계층, 캡슐화 과정에서는 2계층 신호를 01110100, 즉 비트신호로 바꿈.
L2 데이터링크 : 데이터 전송 신호 규정 (MAC 주소 송수신을 위한 데이터링크 프로토콜 선택 - 보통 이더넷), 송수신 시스템 간 데이터에 대한 전기신호를 어떻게 처리할 것인지 규정(헤더와 Tail/Tailer 붙여서 전기신호로 변환되기 전/후 에러 검출/흐름 제어)하는 계층.
L3 네트워크 : 패킷들이 어떻게 네트워크 상에서 라우팅 되는지를 정의 (라우팅 프로토콜로 동적 라우팅하기) , 노드(즉 다음 라우터)에 전송되는 패킷의 흐름을 통제(최적경로 선택???)하고 상태 메시지를 전송(라우팅 시 예를 들어 OSPF면 Hello 패킷 등의 상태 메시지 주고받는 거 이야기하는듯?)하는 계층
L4 트랜스포트 : 네트워크 상에서 송수신 시스템간의 물리적 위치를 논리적인 주소와 이름을 이용하여 연결하는 방식 및 종료하는 방식 등이 정의되는 계층 (HTTP 등의 어플리케이션 계층에서 쓰이는 프로토콜을 번호로 명시하여 도달할 수 있게 함.)
L5 세션 : 표현 계층에서 전송 계층 (프레 to 트랜)으로 진입하기 위해 개념적이고 추상적(즉 논리적)인 인터페이스를 제공하는 계층. (OS 간의 논리적인 연결 담당하고, 어플리케이션 간의 세션을 구축/관리하고 종료합니다.)
L6 프레젠테이션 : 어플리케이션 계층에 의해 생성된 데이터나 payload가 네트워크 상에 전송하기 위해 생산되고 소비되는 데이터 형식이나 형태를 번역하는 방법 등을 정의하는 계층. 즉 데이터의 암호화/복호화(인코드/디코드), 압축/해지 등을 하여 프로그램에서 육안으로 확인 가능하도록 데이터를 구분 및 표현하는 계층. 그래서 데이터가 텍스트인지 이미지인지 등도 판별 가능.
L7 어플리케이션 : 각 응용프로그램 간 정보를 전달하기 위한 계층. 응용 프로세스간의 정보 교환을 담당. 파일 트랜스퍼,액세스 엔 매니퓰레이션, 버츄얼 터미널을 담당. 즉 사용자 인터페이스를 제공하는 계층임. HTTP, FTP, SMTP

MAC 프로토콜 방식	충돌 방어 프로토콜?
1. 파티셔닝 방식 (다중접속)
   - TDMA : 시분할방식 - 사용못하는 시간대 생긴다는 단점
   - FDMA : 주파수분할 방식 - 낭비되는 주파수 대역대 생긴다는 단점
   - CDMA : 코드분할 방식 - 많은 사용자가 동시에 접속하면 혼선이 생긴다는 단저머
2. Random Access 방식
   - *CSMA/CD : 다중 접속에서 혼선된 패킷을 감지하는 CSMA 기능에 충돌 탐지 기능을 추가한 기능. 송신자 A가 수신자 B에게 정보를 전송하고 싶었는데 이미 다른 송신자 C와 통신중이라면, A쪽에서 즉시 통신을 중단하고 정체신호를 보낸다. 그리고 임의의(랜덤의)시간 동안 대기하면서 재전송 준비를 한다. 랜덤시간동안 기다렸다가 재전송했을 때 충돌이 난다면 2배 시간동안 기다렸다가 통신을 한다. 그래서 트래픽이나 시간이 낭비되지 않음. 대신 다른 사용자 패킷이 어떻게 오는지 확인할 뿐. 살아남은 방식.. 단지... 이것도 안쓰임 10기가비트 이더넷에서. 허브에서만 쓰이고 스위치랑 라우터에선 안쓰임. 왜?

이더넷은 CSMA/CD를 적용한 고전적인 방식이다. 그러나 CSMA/CD는 10기가비트 이더넷의 규격에서는 더 이상 사용되지 않는다. 그 이유는 기존 장비를 대체할 스위치 장비의 요구사항 때문이다. 이와 비슷하게, 반이중 상태에서 CSMA/CD의 운용은 기가비트 이더넷의 규격에 정의되어 있으나 실질적으로 적용되지는 않는다. 전이중 상태에서도 데이터의 송수신이 별도로 이루어지며 또한 각 세그먼트 별로 스위치 장비에 직접 연결되기 때문에 CSMA/CD는 사용되지 않는다.
